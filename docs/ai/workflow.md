# Workflow & Reporting Rules

이 문서는 작업 계획 → 승인 → 수정 → 완료 보고 → CHANGELOG 기록까지의
전체 절차를 정의합니다. 파일을 수정하는 모든 작업(`replace`, `write_file` 등)
전후로 이 문서의 절차를 따릅니다.

## Interaction & Workflow Protocol

- **Plan-First Workflow:** 모든 파일 수정이나 시스템 변경 작업 전에는
  반드시 아래 항목을 포함한 [작업 계획]을 한국어로 상세히 보고합니다.
  - **[원인 분석]**
  - **[작업 계획]**
  - **[예상 결과]**
  - **[영향도 분석 (부정적 영향력 유무 및 상세)]**
- **Explicit Approval:** 사용자의 명시적인 승인(Directive)이 있을 때까지
  실제 수정 도구 호출을 대기합니다.
- **Communication Style:** 모든 기술적 분석, 작업 계획, 완료 보고는
  한국어로 작성하며, 간결하고 명확하게 작성합니다. 영문 중복 설명은 생략합니다.

## 작업 후 기록 관리 규칙

모든 작업 완료 후 아래 판단 기준에 따라 변경 유형을 분류하고,
**반드시 도구(`replace`, `write_file` 등)를 사용하여 해당 파일을 직접 업데이트**합니다.

### 자동 기록 및 정돈 원칙

- **자동 업데이트:** 작업 완료 보고 시 `CHANGELOG.md`에 추가할 내용은
  사용자의 별도 요청 없이 자동으로 식별하여 최신 이력(날짜별)에 반영합니다.
- **내용 정돈:** `CHANGELOG.md`를 업데이트할 때, 중복된 내용이 있거나
  분산된 날짜 이력이 있다면 이를 일관성 있게 정돈(병합 및 구조화)하여
  기록의 무결성을 유지합니다.
- **절대 규칙 준수:** 모든 기록은 원본 그대로 유지하며, 기존 이력을 요약,
  삭제 또는 `...` 등의 기호로 생략하는 행위를 절대 금지합니다.
  새로운 작업 내역은 작업한 날짜에만 추가/수정하며, 다른 날짜의 모든 기록은
  원본 그대로 유지해야 합니다.

  ### 변경 유형 판단 기준

| 상황 | CHANGELOG.md | 도메인 SKILL.md + AGENTS.md |
| --- | --- | --- |
| 기능 구현·버그 수정 | ✅ | ❌ |
| 새로운 패턴·규칙 도입 | ✅ | ✅ |
| 기존 규칙 변경·삭제 | ✅ | ✅ |
| 디렉토리 구조 변경 | ✅ | ✅ |
| 기술 스택 추가·제거 | ✅ | ✅ |

> 컨벤션 변경 시 수정 대상은 그 규칙이 실제로 위치한 **단일 파일**입니다.
> (백엔드 규칙 → `.agents/skills/backend/SKILL.md`,
> 프론트엔드 규칙 → `.agents/skills/frontend/SKILL.md`,
> 전역 규칙 → 루트 `AGENTS.md`)
> 같은 내용을 여러 파일에 복붙하지 않습니다.

## 작업 완료 후 필수 출력 형식

작업이 끝나면 반드시 아래 형식으로 변경 유형을 보고합니다.

---

#### 작업 완료 보고

[백엔드/프론트엔드/공통] 제목

**[문제 및 요청 요약]**
작업 내용에 대한 간략한 요약(1~2줄).

**변경 유형**

- [ ] 기능 구현·버그 수정 → CHANGELOG.md 추가
- [ ] 컨벤션 신규·변경·삭제 → 해당 SKILL.md(또는 AGENTS.md) 수정 + CHANGELOG.md 추가

#### CHANGELOG.md 추가 내용

> 아래 내용을 CHANGELOG.md 최상단에 추가했습니다.

```markdown
## [YYYY-MM-DD]

#### Backend (해당 시)
- 작업 내용

#### Frontend (해당 시)
- 작업 내용

#### Convention 변경 (해당 시)
- **[신규·변경·삭제]** `대상 파일` · 변경 내용ㅈ
```

#### 컨벤션 문서 수정 필요 항목 (변경 시)

| 대상 파일 | 변경 전 | 변경 후 |
| --- | --- | --- |
| `파일 경로` | 기존 내용 | 변경 내용 |

> 없으면 "없음"

---

## CHANGELOG에 기록하지 않는 것

다음은 컨벤션 변경이 아니므로 위 절차를 적용하지 않습니다.

- 주석 수정
- 변수명 변경
- Import 정리
- 포맷팅
- 사소한 CSS 수정
